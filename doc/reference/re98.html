Description

This class is a prototype of a vector meeting the hybrid 0-1 interface in LELA. It can be used as a starting-point for those wishing to implement their own hybrid 0-1 vector-types. It cannot be used on its own.

When feasible, suggested implementations of methods are provided.

  2. typedef std::iterator_traits< iterator >::value_type value_type;

    This should in general be an STL-pair of an integral block-index-type and a word.

  3. typedef VectorStorageTypes::Generic StorageType;

    Use Real if the vector represents a live array of index-word pairs in memory, Transformed if it transforms a pair of vectors into a vector of pairs (as with HybridVector), or Generic if none if these is true.

  4. typedef SparseSubvector< const Hybrid01VectorInterface, VectorRepresentationTypes::Hybrid01 > SubvectorType;

    The subvector-type must support a construct of the form Subvector (v, start_index, end_index) where v is the parent-vector and start_index and end_index are indices of the beginning and end of the subvector.

    Normally it suffices to use SparseSubvector for this type as given here.

    Since SparseSubvector does not support non-const arbitrary (i.e. not word-aligned) subvectors of hybrid vectors, we set SubvectorType to be identical to ConstSubvectorType. It is then an error to construct an arbitrary subvector of a hybrid vector and then attempt to modify it (e.g. with push_back).

  5. typedef SparseSubvector< const Hybrid01VectorInterface, VectorRepresentationTypes::Hybrid01 > ConstSubvectorType;

    Normally it suffices to use SparseSubvector for this type as given here.

  6. typedef SparseSubvector< Hybrid01VectorInterface, VectorRepresentationTypes::Hybrid01 > AlignedSubvectorType;

    In the case of hybrid vectors, this usually means aligning the subvector along a word-boundary.

    Normally it suffices to use SparseSubvector for this type as given here.

  7. typedef SparseSubvector< const Hybrid01VectorInterface, VectorRepresentationTypes::Hybrid01 > ConstAlignedSubvectorType;

    Normally it suffices to use SparseSubvector for this type as given here.
