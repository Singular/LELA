Description

BitVector provides an additional iterator, word_iterator, that gives the bits in compact 32-bit words, so that vector operations may be done in parallel. It is similar to the STL bit_vector except that it provides the aforementioned additional iterator.

Notes about semantics:

  • word_iterator is guaranteed to iterate up to the *penultimate* word, not the last word

  • back_word gives a reference to the last word. It is not necessarily *(word_end () - 1)

  • Therefore, code working on BitVectors (as well as BitSubvectors) should iterate through the word_iterator and then do the same operation on back_word ().
