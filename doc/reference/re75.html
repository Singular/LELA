Description

This implements the naive Gaussian elimination algorithm. No attempt is made to take advantage of fast matrix-multiplication, cache-optimisation, or other things.

  1. template<typename Matrix1, typename Matrix2> 
  void RowEchelonForm(Matrix1 & A, Matrix2 & L, Permutation & P, 
                      size_t & rank, Element & det, bool reduced = false, 
                      bool compute_L = true, size_t start_row = 0) const;
    Compute the (reduced or non-reduced) row-echelon form of a matrix.

    At conclusion, the parameters will have the property that A_out=LPA_in, where A_out is the matrix A at output and A_in is the matrix A at input. A_out is in reduced row-echelon form, L is lower triangular, and P is a permutation.

    In comparison with

    See Also:

    RowEchelonForm, this version does not take advantage of fast matrix-multiplication and does not use a divide-and-conquer method. It also modifies the input-matrix A.

    The pivot-strategy is to find the row with the fewest elements. This seems to be the only sensible approach, given that we are not allowed to permute columns.

    Parameters:

    A

    The sparse matrix whose reduced row-echelon form to compute. Will be replaced by its reduced row-echelon form during computation.

    L

    The dense matrix into which to store the matrix L as defined above. Should be of size n x n, with n the row-dimension of A.

    P

    The permutation into which to store the permutation P as defined above.

    compute_L

    True if the matrix L should be computed. If false, then L is ignored.

    det

    A ring-element into which to store the computed determinant of the submatrix of A formed by taking pivot-rows and -columns.

    rank

    An integer into which to store the computed rank of A.

    reduced

    True if the routine should compute the reduced row-echelon form and false if it should only compute the (non-reduced) row-echelon form.

    start_row

    Start at this row. Intended for internal use.
