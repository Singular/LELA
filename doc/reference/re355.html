Description

This class is a prototype of a vector meeting the sparse 0-1 interface in LELA. It can be used as a starting-point for those wishing to implement their own sparse 0-1 vector-types. It cannot be used on its own.

When feasible, suggested implementations of methods are provided.

Sparse01VectorInterface public types

  1. typedef std::iterator_traits< iterator >::value_type value_type;

    This should in general be an integral index-type

  2. typedef VectorStorageTypes::Generic StorageType;

    Use Real if the vector represents a live array of indices in memory, Generic if not.

  3. typedef SparseSubvector< Sparse01VectorInterface, VectorRepresentationTypes::Sparse01 > SubvectorType;

    The subvector-type must support a construct of the form Subvector (v, start_index, end_index) where v is the parent-vector and start_index and end_index are indices of the beginning and end of the subvector.

    Normally it suffices to use SparseSubvector for this type as given here.

  4. typedef SparseSubvector< const Sparse01VectorInterface, VectorRepresentationTypes::Sparse01 > ConstSubvectorType;

    Normally it suffices to use SparseSubvector for this type as given here.

  5. typedef SparseSubvector< Sparse01VectorInterface, VectorRepresentationTypes::Sparse01 > AlignedSubvectorType;

    In nearly all cases, it should be identical to Subvector.

    Normally it suffices to use SparseSubvector for this type as given here.

  6. typedef SparseSubvector< const Sparse01VectorInterface, VectorRepresentationTypes::Sparse01 > ConstAlignedSubvectorType;

    Normally it suffices to use SparseSubvector for this type as given here.

Sparse01VectorInterface public public data members

  1. static const int align;

    In nearly all cases -- and particularly when SubvectorType and AlignedSubvectorType are the same, it should be one.
